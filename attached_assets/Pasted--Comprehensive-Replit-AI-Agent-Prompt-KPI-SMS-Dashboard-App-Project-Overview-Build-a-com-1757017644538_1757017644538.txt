# Comprehensive Replit AI Agent Prompt: KPI SMS Dashboard App

## 🎯 Project Overview
Build a complete KPI SMS automation platform similar to CEOTXT that sends weekly SMS summaries of 7 key business metrics to executives. This should be a full-stack web application with both a user interface for configuration and automated SMS delivery.

## 🏗️ Technical Architecture Required

### Backend: Python + Flask
- **Framework**: Flask web application with proper routing
- **Database**: SQLite/PostgreSQL for storing user data, KPI configurations, and historical metrics
- **Background Jobs**: Python `schedule` library for weekly automation
- **APIs**: RESTful endpoints for frontend interaction
- **Error Handling**: Comprehensive try/catch blocks and logging

### Frontend: HTML/CSS/JavaScript (or React if preferred)
- **Dashboard**: Clean, modern interface for viewing KPIs and trends
- **Configuration Pages**: Forms to set up KPIs, recipients, and data sources
- **Authentication**: User login/logout system
- **Responsive Design**: Works on desktop and mobile

### External Integrations
- **SMS Service**: Twilio API integration (account_sid and auth_token via environment variables)
- **Data Sources**: At least 3 API integrations from: Stripe, HubSpot, QuickBooks, Google Analytics
- **Security**: Environment variables for all API keys

## 📊 Core Features to Implement

### 1. User Management System
- User registration and login with password hashing
- Company/organization management (multi-tenant ready)
- Role-based permissions (admin, viewer)

### 2. KPI Configuration Dashboard
- **Available KPIs to choose from**:
  - Revenue (weekly total and % change)
  - Customer Acquisition Cost (CAC)
  - Churn Rate
  - Net Promoter Score (NPS)
  - Cash Flow/Bank Balance
  - Gross Margin
  - Sales Pipeline Value
- **KPI Setup Form**: Allow users to select 7 KPIs and configure data sources
- **Data Source Configuration**: Forms to input API credentials for each integration

### 3. SMS Recipients Management
- Add/remove phone numbers for SMS delivery
- Test SMS functionality
- SMS delivery history and status tracking

### 4. Data Collection Engine
- **Stripe Integration**: Revenue, subscription metrics, customer counts
- **HubSpot Integration**: Deals, pipeline, customer data
- **QuickBooks Integration**: Financial data, expenses, profit margins
- **Manual Input Option**: For KPIs that can't be automatically collected

### 5. Weekly SMS Automation
- **Scheduler**: Run every Friday at 8:00 AM (configurable)
- **Message Format**: 
```
W18 Summary
• Revenue $612k ▲3%
• CAC $186 ▼4%
• Churn 1.9% ▲0.2
• NPS 62 ▲1
• Cash $1.9M
• Gross Margin 63% ▲2%
• Pipeline $3.1M ▲5%
```
- **Error Notifications**: Email alerts if SMS delivery fails

### 6. Historical Data & Analytics
- Store weekly KPI snapshots in database
- Trend analysis and charts
- Export capabilities (CSV/PDF)
- Performance comparisons (week-over-week, month-over-month)

### Database Schema
```sql
-- Users and Companies
CREATE TABLE companies (id, name, created_at);
CREATE TABLE users (id, company_id, email, password_hash, role);

-- KPI Configuration
CREATE TABLE kpi_definitions (id, company_id, kpi_name, data_source, is_active);
CREATE TABLE data_source_configs (id, company_id, source_name, api_credentials_encrypted);

-- Historical Data
CREATE TABLE kpi_snapshots (id, company_id, week_number, year, kpi_name, value, previous_value, change_percent, created_at);

-- SMS Management
CREATE TABLE sms_recipients (id, company_id, name, phone_number, is_active);
CREATE TABLE sms_delivery_log (id, company_id, recipient_id, message_content, status, sent_at);
```

### **Fallback & Error Handling Strategy**

#### **Smart Defaults for Missing Data**:
- If API fails, use last known value with "⚠️" indicator
- Allow manual override for any KPI during outages
- Email notifications when data collection fails
- Graceful degradation (send SMS with available metrics only)

#### **Data Validation Rules**:
- Revenue must be positive numbers
- Percentages must be 0-100 (or 0-1 for decimals)
- Growth rates flagged if >50% change week-over-week
- Missing data clearly marked in SMS format
